# 📝 Wishlist API (Mini-Backend)

RESTful API для сервісу **«Список бажань»**, побудований на архітектурі **Controller-Service-Model**. Дозволяє користувачам керувати своїми бажаннями, встановлювати пріоритети та додавати зображення до кожного запису.

---

## 🚀 Технологічний стек

| Категорія     | Технологія                  |
|---------------|-----------------------------|
| Runtime       | Node.js                     |
| Framework     | Express                     |
| Database      | MySQL (через Sequelize ORM) |
| Validation    | Zod                         |
| File Upload   | Multer                      |
| Testing       | Jest                        |

---

## 🛠 Налаштування та запуск

### 1. Встановіть залежності

```bash
npm install
```

### 2. Налаштуйте середовище

Створіть файл `.env` у кореневій директорії та вкажіть дані для підключення до MySQL:

```env
DB_NAME=myshop_db
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_HOST=localhost
DB_PORT=3306
```

### 3. Запустіть сервер

```bash
npm run dev
```

## 📂 Структура бази даних

| Таблиця       | Опис                                                              |
|---------------|-------------------------------------------------------------------|
| `Users`       | Зберігає інформацію про власників списків бажань                 |
| `Tasks`       | Містить дані про подарунки/бажання, їх статус та пріоритетність  |

## 📖 Документація API

### 👤 Користувачі (Users)

#### `POST /users` — Реєстрація нового користувача

**Body (JSON):**
```json
{
  "name": "Olya",
  "email": "test@test.com"
}
```

---

### 🎁 Бажання (Tasks)

#### `POST /tasks` — Створення нового бажання

**Body (JSON):**
```json
{
  "title": "iPhone 17",
  "priority": 5,
  "userId": 1
}
```

---

#### `GET /tasks` — Отримання списку бажань

Підтримує фільтрацію, сортування та пагінацію.

**Query параметри:**

| Параметр   | Значення            | Опис                          |
|------------|---------------------|-------------------------------|
| `status`   | `open` \| `done`    | Фільтр за статусом            |
| `priority` | `1..5`              | Фільтр за пріоритетом         |
| `search`   | рядок               | Пошук по назві або описі      |
| `sort`     | `createdAt` \| `priority` | Поле для сортування    |
| `order`    | `asc` \| `desc`     | Напрямок сортування           |
| `page`     | число               | Номер сторінки (пагінація)    |
| `limit`    | число               | Кількість записів на сторінці |

**Приклад запиту:**
```
GET /tasks?status=open&priority=5&sort=createdAt&order=desc&page=1&limit=10
```

---

#### `GET /tasks/:id` — Деталі конкретного бажання

Повертає повну інформацію про бажання, включаючи прикріплені фото.

---

#### `PATCH /tasks/:id` — Оновлення бажання

**Body (JSON):**
```json
{
  "status": "done"
}
```

Можна оновити `status` та/або `priority`.

---

#### `DELETE /tasks/:id` — Видалення бажання

---

### 📎 Вкладення (Attachments)

#### `POST /tasks/:id/attachments` — Завантаження зображень

Дозволяє завантажити від **1 до 3 зображень** для конкретного бажання.

**Body:** `form-data`, поле `files` (тип: File).

---

#### `GET /tasks/:id/attachments` — Список файлів бажання

Повертає список усіх файлів, прикріплених до вказаного бажання.

## 📁 Структура проєкту

```
├── controllers/       # Обробники HTTP-запитів
├── services/          # Бізнес-логіка
├── models/            # Sequelize-моделі (Users, Tasks, Attachments)
├── routes/            # Маршрути Express
├── middlewares/       # Валідація (Zod), обробка файлів (Multer)
├── uploads/           # Завантажені файли
├── tests/             # Unit-тести (Jest)
├── .env               # Змінні середовища (не комітити!)
└── app.js             # Точка входу
```

## 🔒 Примітки

- Файл `.env` **не повинен потрапляти до репозиторію** — додайте його до `.gitignore`.
- Папка `uploads/` може бути додана до `.gitignore`, якщо файли не зберігаються у репозиторії.
